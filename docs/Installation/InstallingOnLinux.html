Installing Scheduler on a Linux System

To run Scheduler on a Linux system, you will need to do the following:

  1. Make sure that Perl is installed on your system.
  2. Download the Scheduler code to your system.
  3. Install the needed Perl modules to your system.

Scheduler uses Gmail to send out email notifications to volunteers, and uses Twilio to send out text messages to volunteers. If you wish to make use of these features

For Gmail you will need to:
  1. Set up a Google account.
  2. Create a Google project.
  3. Get the Google API key for that project.
For Twilio you will need to:
  1. Set up a Twilio account.
  2. Get the Twilio information.

Scheduler runs as a web server. When it is running, you should be able to access it from any machine that is on the same local network as the system where it is running. If you wish to be able to access it from outside your office (or allow volunteers to access it to make updates to their available dates) then you will need to make your local machine visible on the internet.

Make sure that Perl is installed on your system

Perl is probably already installed on your system, as it is installed by default on most unix distributions. To see if Perl is installed on your system, open a terminal session, and at the prompt, type: perl -v If you see something that starts off like: This is perl 5, version 34, subversion 0 (v5.34.0) then you have Perl installed on your system. (The version number doesn't have to match.)

If, on the other hand, you see something like: Command 'perl' not found then Perl is not installed on your system, and you need to install it before you can run Scheduler.

Installing Perl

To install Perl on your system, the recommended method is to use the package manager associated with your system. The two most common are yum, which is used on RedHat based systems, like RHEL, Fedora and CentOs, and apt, which is used on Debian based systems like Debian, Ubuntu, and Mint. (It is worth mentioning that instead of apt, your system might have apt-get installed. Apt is a newer version, but apt and apt-get have similar functionality.) Note that to use either apt or yum to install packages, you will either need to be logged in as the root user, or you will need to run each command using sudo. The examples here will use sudo. If you are unsure if your system uses apt or yum you may simply try typing each command at the command prompt.

Using yum

First it is recommended that you update the list of the latest versions of software that is stored on your computer. To do so, open a terminal session and type: sudo yum check-update Once that has been done, you can then install the latest packaged version of Perl by typing the following: sudo yum install perl This will direct the yum program to download Perl and any additional required libraries that are needed by Perl. Once the packages are downloaded they will be installed, along with any of the needed dependencies.


Using apt

If your system has apt-get installed, you may simply substitute "apt-get" for "apt" in these instructions. Before installing a new package, it is recommended that you first update your computer's list of available packages and versions. To do so, open a terminal session and type: sudo apt update The system will reach out to other systems on the internet to get a list of the latest package versions that are currently available for your system. When that is finished, you can then install Perl by typing:

sudo apt install perl

Install the needed Perl modules to your system

In addition to Perl itself, your system will also need to have some additional Perl modules. Perl modules are reusable components that are not part of the base installation that make it easier to carry out specific tasks. The first step is to make sure you have a Perl tool named cpanm.

cpanm

To install the needed Perl modules, you will need to use a Perl tool named cpanm. Much like apt or yum, cpanm is a package manager that handles the work of downloading, testing, and installing Perl modules. To see if it is already on your system, open a terminal session and type:

cpanm If you see something like: Usage: cpanm [options] Module [...]
Try `cpanm --help` or `man cpanm` for more options. then it is already installed on your system. If not, you will need to install cpanm.

Installing cpanm

Installing cpanm is easy, but you first need to see if your system includes a make command. You can do this by opening a terminal session and typing: make If you see something like: make: *** No targets specified and no makefile found. Stop. then it is already installed. If not, use either apt or yum to install make by typing either:

  • sudo apt install make
  • sudo yum install make
as appropriate for your system.

Once make is installed on your system, you are ready to install cpanm. To do so, in a terminal session, type: curl -L https://cpanmin.us | sudo perl - App::cpanminus Cpanm should now be installed on your system.
